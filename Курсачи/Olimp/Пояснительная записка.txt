Первое создаем базу данных
скрипт ее
CREATE TABLE [dbo].[Student] (
    [Id]      INT         NOT NULL,
    [FIO]     NCHAR (120) NOT NULL,
    [Score]   INT         NULL,
    [Subject] NCHAR (50)  NULL,
    PRIMARY KEY CLUSTERED ([Id] ASC)
);

Затем по данной базе данных строим модель. Модель содержит класс олимпиадников (Student) и класс Model1, он содержит данные для подключения и выборки контекста.
Данную модель мы делаем по принципу code second, а тоесть, вначале бд, потом классы, классы были автоматически созданы с помощью ADO.
До подключения кладем нашу бд  по даннолму пути C:\Users\Имя\AppData\Local\Microsoft\Microsoft SQL Server Local DB\Instances\mssqllocaldb\Students.mdf,
или лучше указываем путь, который будет удобен вам.
Когда делаем подключения в ADO, обязательно в сервере пишем сервер -- (localdb)\MSSQLLocalDB и название нашей таблицы.
Дальше работа с бд будет происходить с Entity fraem work.
Первая форма ввода loginform содержит 2 поля, логин и пароль и простую захардкоденую проверку на пользователя, их введено в систему 2, один с доступом к изменениям, второй только с просмотром данных.
Главная форма это Form1, в конструкторе формы добавили combox для сортировки посредством sql запросов и две кнопки, для сохранения и добавления олимпиадников.
Также добавлено меню, для открытия о программе, там тектст пустой и путь захардкоден, не перемещайте текстовый документ.
И главное на форме это - DataGridView, через пИсточник данных мы связываем его с нашим классов Student и создаем подключение в коде указано в коментариях как оно происходит.
